Tooltip row — §24 fixed-dark treatment · v2 (render before ratify)

The one polarity exception (§6): tooltips are fixed-dark labels, the SAME dark chip on every theme. v2 changes (Ari): the separating edge is now alpha-white (self-adjusting — a light rim on dark chrome, invisible on white); the trigger gap is tightened to a real 5px offset (v1 mocked it too loose, which unfairly punished the no-arrow read); and key commands render as an on-dark Kbd keycap reusing the tooltip's own material (§D). D · Kbd inside the tooltip — on-dark keycap

The consistency ask: key commands use the Kbd primitive. Kbd is normally theme-following (would go invisible on a dark theme inside a dark tooltip), so inside the tooltip it renders on-dark — an alpha-white face + edge + the tooltip's light ink. Shown on paper · graphite · carbon: because the tooltip is fixed-dark, the keycap looks identical on all three — that's the consistency win. Rightmost: plain-text shortcut (v1) for contrast.
